How It Works

This tool calculates the project duration based on the tasks you enter, their durations, and dependencies. Here's a brief overview of the data structures and algorithms used:

  • Tasks: Represented by a JavaScript object with properties for ID, duration, dependencies, earliest start, latest start, and time reserve.
  • Dependencies: A list of task IDs that the current task depends on. The task can only start once all its dependencies have finished.
  • Calculation Algorithm: The algorithm first calculates the earliest start and finish times for each task based on its dependencies. Then, it calculates the latest start and finish times to determine the slack or time reserve for each task. This helps in identifying critical path tasks that cannot be delayed without affecting the overall project duration.

By entering the tasks, durations, and dependencies, the tool visualizes the project timeline and highlights the flexibility you have with non-critical tasks.
